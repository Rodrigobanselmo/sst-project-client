# CSelect

TextFields allow users to take write text with a single the keyboard. Supports all default HTML input properties.

## Import

```tsx
import { CSelect } from '@camel/camel-react';
```

## Usage

```tsx
<CSelect label="Primary" />
```

### Options

In the options prop ypu can pass an array of strings or an object with the following properties: "value", "content". You can also pass an array of objects with any properties you want as long as you pass the correspondent fields to the optionsFieldName prop.

<Canvas>
  <Story name="Options Prop">
    <div style={containerStyles}>
      <CSelect
        label="Options"
        options={[
          { value: 1, content: 'Jupter' },
          { value: 2, content: 'Sapturne' },
          { value: 3, content: 'Uranio' },
        ]}
      />
    </div>
  </Story>
</Canvas>

### Options FieldName

With this you can use different field name for the value and content of the select menu itens, by passing an options array with any field name and specifing the field name on the optionsFieldName prop.

<Canvas>
  <Story name="Options FieldName">
    <div style={containerStyles}>
      <CSelect
        label="OptionsFieldName"
        optionsFieldName={{ valueField: 'id', contentField: 'planet' }}
        options={[
          { id: 1, planet: 'Jupter' },
          { id: 2, planet: 'Sapturne' },
          { id: 3, planet: 'Uranio' },
        ]}
      />
    </div>
  </Story>
</Canvas>

### Disabled

If true, the component is disabled.

<Canvas>
  <Story name="Disabled">
    <CSelect options={options} label="Disabled" disabled />
  </Story>
</Canvas>

### Required

If true, the label is displayed as required and the input element is required.

<Canvas>
  <Story name="Required">
    <CSelect options={options} label="Required" required />
  </Story>
</Canvas>

### Empty Item

If true, the label is displayed as required and the input element is required.

<Canvas>
  <Story name="EmptyItem">
    <CSelect options={options} label="EmptyItem" emptyItem={<em>empty</em>} />
  </Story>
</Canvas>

### Error

If true, the label is displayed in an error state.

<Canvas>
  <Story name="Error">
    <CSelect
      label="Error"
      options={options}
      error
      helperText="error helper text"
    />
  </Story>
</Canvas>

### Loading

If true, it adds a circular loading indicator to the end of the input. You can pass a circularProps that accepts any CircularProgress props.

<Canvas>
  <Story name="Loading">
    <div style={containerStyles}>
      <CSelect label="Loading" options={options} loading />
      <CSelect
        label="Loading"
        options={options}
        loading
        size="small"
        circularProps={{
          color: 'secondary',
          style: { opacity: 0.2 },
          size: 15,
        }}
      />
    </div>
  </Story>
</Canvas>

### Multiple

If true, you will be able to select multiple values. The "value" and "onChange" props will be required and "value" has to be an array of strings

<Canvas>
  <Story name="Multiple">
    <div style={containerStyles}>
      <CSelect
        multiple
        width={220}
        value={['Oliver Hansen', 'Van Henry']}
        options={names}
        label="Default"
      />
    </div>
  </Story>
</Canvas>
